# 1-100 专注力训练游戏 - UI 设计说明

## 设计理念

参考"羊了个羊"等热门小游戏的交互模式，采用：
- 🎨 **温暖渐变背景** - 不同界面使用不同色调的渐变
- 🎯 **大按钮设计** - 易于点击，符合移动端操作习惯
- ✨ **卡片式布局** - 清晰的层次感和阴影效果
- 🌈 **色彩区分** - 不同模式和难度使用不同颜色标识

## 界面流程

```
开始界面 → 模式选择 → 难度选择 → 游戏界面 → 结果界面
   ↑                                              ↓
   └──────────────── 返回首页 ←──────────────────┘
```

## 界面详情

### 1. 开始界面 (Start Screen)
- **背景**: 温暖的橙黄色渐变 (#FFE5B4 → #FFC48C)
- **元素**:
  - 大标题 "1 → 100"
  - 副标题 "专注力挑战"
  - 开始游戏按钮 (红色 #FF6B6B)
  - 游戏说明文字
- **装饰**: 半透明白色圆点装饰

### 2. 模式选择界面 (Mode Select)
- **背景**: 清新的蓝色渐变 (#E3F2FD → #BBDEFB)
- **元素**:
  - 标题 "选择游戏模式"
  - 明亮模式卡片 (绿色 #4CAF50)
    - 图标: ☀️
    - 说明: 清晰可见所有数字
  - 黑暗模式卡片 (紫蓝色 #5C6BC0)
    - 图标: 🌙
    - 说明: 手电筒照亮，挑战更高难度
  - 返回按钮

### 3. 难度选择界面 (Difficulty Select)
- **背景**: 温暖的橙色渐变 (#FFF3E0 → #FFE0B2)
- **布局**: 2×2 网格布局
- **难度选项**:
  - 自由模式 (绿色 #66BB6A) - 无时间限制
  - 1分钟 (橙色 #FFA726) - 限时60秒
  - 2分钟 (深橙 #FF7043) - 限时120秒
  - 3分钟 (红色 #EF5350) - 限时180秒
- **返回按钮**

### 4. 游戏界面 (Game Screen)
- 保持原有的游戏渲染
- HUD 显示:
  - 目标数字
  - 计时器
  - 错误次数
  - 连击数（黑暗模式）
  - 进度条

### 5. 结果界面 (Result Screen)
- **背景**: 
  - 胜利: 绿色渐变 (#E8F5E9 → #C8E6C9)
  - 失败: 红色渐变 (#FFEBEE → #FFCDD2)
- **元素**:
  - 结果标题 (🎉 完成！/ ⏰ 时间到)
  - 成绩卡片:
    - 完成数量 (X/100)
    - 用时
    - 错误次数
    - 最高连击
  - 再来一局按钮 (绿色)
  - 返回首页按钮 (蓝色)

## 交互特性

### 按钮效果
- ✅ 圆角设计 (border-radius: 15-25px)
- ✅ 阴影效果 (shadowBlur: 8-15px)
- ✅ 清晰的点击区域
- ✅ 颜色区分不同功能

### 卡片设计
- ✅ 白色背景 + 阴影
- ✅ 左侧大图标
- ✅ 右侧文字说明
- ✅ 箭头指示可点击

### 视觉层次
1. **背景层**: 渐变色 + 装饰圆点
2. **内容层**: 卡片和按钮
3. **文字层**: 标题和说明

## 颜色方案

### 主色调
- 🔴 红色系: #FF6B6B, #EF5350 (强调、警告)
- 🟢 绿色系: #4CAF50, #66BB6A (成功、确认)
- 🔵 蓝色系: #2196F3, #5C6BC0 (信息、次要)
- 🟠 橙色系: #FFA726, #FF7043 (警示、中等)

### 背景色
- 温暖橙: #FFE5B4 → #FFC48C
- 清新蓝: #E3F2FD → #BBDEFB
- 柔和橙: #FFF3E0 → #FFE0B2
- 成功绿: #E8F5E9 → #C8E6C9
- 失败红: #FFEBEE → #FFCDD2

## 字体规范

- **标题**: PingFang SC / Microsoft YaHei (粗体)
- **正文**: PingFang SC / Microsoft YaHei (常规)
- **大小**:
  - 主标题: 12% 屏幕宽度
  - 副标题: 4.5-7% 屏幕宽度
  - 按钮文字: 4-5% 屏幕宽度
  - 说明文字: 3.5-4% 屏幕宽度

## 响应式设计

所有尺寸基于屏幕宽度百分比计算，确保在不同设备上都有良好的显示效果：
- 按钮宽度: 60-80% 屏幕宽度
- 卡片宽度: 80% 屏幕宽度
- 间距: 基于屏幕尺寸动态计算

## 用户体验优化

1. **清晰的导航**: 每个界面都有返回按钮
2. **即时反馈**: 按钮点击立即响应
3. **视觉引导**: 使用颜色和图标引导用户
4. **信息层次**: 重要信息突出显示
5. **易于操作**: 大按钮，易于点击

## 技术实现

- 使用 Canvas 2D API 绘制所有UI元素
- 圆角矩形使用 quadraticCurveTo 实现
- 渐变背景使用 createLinearGradient
- 阴影效果使用 shadowColor/shadowBlur
- 点击检测通过坐标范围判断
